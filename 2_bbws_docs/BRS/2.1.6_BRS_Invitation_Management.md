## 1. Executive Summary

### 1.1 Purpose

The **Invitation Management** capability enables organisation administrators to invite users to join their organisation within the BBWS platform. Users receive secure invitation links to join organisations with pre-assigned roles.

### 1.2 Key Benefits

| Benefit | Business Impact |
|---------|-----------------|
| **Self-Service Invitations** | Admins grow teams without support tickets |
| **Secure Onboarding** | Unique tokens prevent unauthorized access |
| **Role Assignment** | Users join with correct permissions |
| **Audit Trail** | Track all invitation activity |

---

## 2. Business Problem

| Problem | Impact |
|---------|--------|
| **Manual User Addition** | Support must add users to organisations |
| **No Invitation Tracking** | Cannot verify invitation status |
| **Role Confusion** | Users added without proper permissions |

| Metric | Before | After | Improvement |

## 4. Scope Definition

### 4.1 In Scope

| Capability | Description |
|------------|-------------|
| **View Invitation** | See invitation details from link |
| **Accept Invitation** | Join organisation with assigned role |
| **Decline Invitation** | Reject invitation |
| **Expiry Handling** | Expired invitations rejected |

### 4.2 Out of Scope

| Capability | Handled By |
|------------|------------|
| Creating invitations | Admin Portal (2.4) |
| Organisation management | Tenant Management (2.5) |
| User roles | Tenant Management (2.5) |

---

## 5. Business Capabilities

### 5.1 Epic 1: Invitation Response

| ID | As a... | I want to... | So that... |
|----|---------|--------------|------------|
| US-2.1.6-001 | Invited User | view invitation details | I know which organisation invited me |

---

## 6. Business Rules

| Rule ID | Rule | Rationale |
|---------|------|-----------|
| BR-INV-001 | Invitations expire after 7 days | Security |
| BR-INV-002 | Each invitation token is single-use | Prevent sharing |
| BR-INV-003 | Accepted invitations cannot be reused | Security |
| BR-INV-004 | Declining notifies organisation admin | Transparency |
